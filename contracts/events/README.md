# Event contracts

Source of truth for every Kafka message on the backbone. Files are generated by
`scripts/gen-event-schemas.py` — edit the generator, not the JSON.

## Conventions

- **Topic naming:** `<domain>.<aggregate>.<event>.v<N>` — lowercase, dots, versioned.
  Topics are never renamed; breaking payload changes bump `v<N>` (new topic + new subject).
- **Message key:** aggregate id (`payment_id`, `account_id`, `user_id`) — preserves
  per-aggregate ordering within a partition.
- **Envelope:** every event carries `event_id` (uuidv7 — the consumer dedup key),
  `event_type` (== topic), `schema_version`, `occurred_at`, `tenant_id`,
  `correlation_id` (W3C trace id), `causation_id`, `idempotency_key`, `payload`.
- **Wire format:** Confluent framing (magic byte 0x00 + 4-byte schema id + JSON),
  produced/consumed through the Apicurio ccompat endpoint
  (`http://apicurio-registry:8080/apis/ccompat/v7`), subject `"<topic>-value"`,
  `TopicNameStrategy`. Never mix raw JSON and framed messages on one topic.
- **Delivery:** at-least-once. Consumers MUST be idempotent — dedupe on `event_id`
  via a `processed_events` table written in the same DB transaction as the
  business write.
- **DLQs:** per consumer group: `<group>.<topic>.dlq`. Original payload untouched;
  failure metadata in headers (`x-exception`, `x-original-topic`,
  `x-original-partition`, `x-original-offset`, `x-failed-at`, `x-consumer-group`).
  Non-retryable poison → DLQ after 3 attempts; retryable errors use in-place
  backoff. Alert on DLQ depth > 0.
- **Money:** integer `amount_minor_units` + ISO-4217 `currency_code`, everywhere.
